  that the water die panel would reappear when players changed shape dice
  even after they had already selected a building type for the water die.

  Changes Made

  1. Modified DiceUIManager.cs - UpdateWaterDieUI() method (line 221-243)

  - Updated the logic to only show the water die panel when:
    a. A water die is currently selected AND
    b. No building type has been chosen yet for that water die
  - The panel now stays hidden when a building type has already been
  selected, even when shape dice are changed
  - The previously selected building type persists across shape changes

  2. Updated documentation in current-functionality.md

  - Added note about the fix in the "Basic UI implemented" section
  - Documented that the water die panel now only appears when water die is
  selected with no building type chosen yet

  Key Behavior Changes

  1. Before: Water die panel appeared every time water die was selected,
  regardless of whether a building type had been chosen
  2. After: Water die panel only appears when:
    - Water die is selected AND
    - No building type has been chosen yet
    - Once a building type is chosen, the panel stays hidden for the rest of
   that turn
  3. Building Type Persistence: When a player selects a building type for a
  water die and then changes shape dice, the previously chosen building type
   is retained and used for the new shape

  Technical Details

  The fix leverages the existing waterDieChosenBuildingType state in
  DiceManager.cs:
  - DiceManager.IsWaterDieChosenBuildingTypeSet() checks if a building type
  has been selected
  - DiceManager.GetBuildingTypeForShape() returns the chosen building type
  when water die is selected
  - The water die panel UI now respects this state and only appears when
  needed

  The solution maintains all existing functionality while fixing the
  undesired panel reappearance behavior described in the request.

‚óŹ I've implemented the requested fix so the water die panel appears when the
   water die is clicked in the dice panel, regardless of whether it's
  already selected or has a building type chosen.

  Changes Made

  1. Modified DiceUIManager.cs

  - Added a waterDieClickedThisFrame flag to track when the water die is
  specifically clicked
  - Updated OnBuildingDieClicked() to set the flag when a water die is
  clicked
  - Modified UpdateWaterDieUI() to show the panel when either:
    a. No building type has been chosen yet, OR
    b. The water die was clicked this frame (allowing users to change their
  building type choice)
  - Added flag resets in OnWaterDieBuildingTypeClicked(), OnDiceRolled(),
  and OnSelectionCleared() for proper state management

  2. Updated Documentation in current-functionality.md

  - Updated the water die panel behavior description to reflect the new
  logic

  New Behavior

  1. Water die clicked ‚Üí Panel appears (even if already selected and
  building type chosen)
  2. Shape dice changed while water die selected ‚Üí Panel stays hidden if
  building type already chosen
  3. Building type selected ‚Üí Panel hides (as before)
  4. Non-water building die clicked ‚Üí Panel hides, water die deselected
  5. Dice rolled/new turn ‚Üí Panel hides, state reset

  Key Logic

  - The water die panel now serves two purposes:
    - Initial building type selection (when no type chosen)
    - Building type change (when user clicks water die again)
  - The previously selected building type persists across shape changes
  unless the user explicitly changes it
  - The panel only appears automatically when first selecting water die with
   no building type chosen; otherwise it requires a click on the water die
  to appear

  This gives users the flexibility to change their building type choice
  while maintaining the previous fix that prevented the panel from appearing
   unnecessarily during shape changes.
